# Module 10: Product Prototyping

## Інструменти

- **Rapid prototyping** — швидке створення варіантів для перевірки ідеї, UX або технічної можливості; мінімум часу, максимум навчання.
- **Класифікація прототипів: Alpha, Beta, Pilot** — Alpha: внутрішня перевірка; Beta: обмежена зовнішня аудиторія; Pilot: обмежений реліз у реальному середовищі з метриками.
- **Артефакти прототипування** — макети (Figma, Sketch), інтерактивні прототипи, click-through flows, MVP-версії в коді.
- **Wireframing** — низькоточкові схеми структури та потоків; швидко, без відволікань на візуал; основа для обговорення з командою та стейкхолдерами.
- **3D printing / фізичні прототипи** — коли продукт має hardware або фізичний компонент; швидка перевірка форми, розміру, взаємодії.
- **Best practices** — чітна мета прототипу (що перевіряємо); правильна аудиторія тестів; не переробляти прототип у продакшн без окремого рішення.

## Висновки (запам’ятати та використовувати)

1. **Прототип — для навчання, не для продакшну:** мета — перевірити гіпотезу (UX, техніка, сприйняття); не треба робити його “ідеальним”.
2. **Alpha → Beta → Pilot — етапи зниження ризику:** кожен етап розширює аудиторію і підвищує вимоги до стабільності; не пропускай етапи для критичних продуктів.
3. **Wireframes економлять час:** спочатку узгодь структуру і потік через wireframes, потім деталізуй дизайн; зміни на етапі wireframe дешевші.
4. **Прототип має відповідати питанню:** якщо перевіряєш “чи зрозуміло?”, достатньо статичного макету; якщо “чи працює інтеграція?” — потрібен технічний прототип.
5. **Після тесту прототипу — рішення: ітерація, відміна або масштабування:** якщо нічого не змінюється після тесту, прототипування марне; завжди фіксуй висновки та наступні кроки.
